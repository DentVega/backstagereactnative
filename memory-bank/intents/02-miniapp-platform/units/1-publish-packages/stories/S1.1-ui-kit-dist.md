# S1.1 — ui-kit: build a dist + publishConfig

**Como** plataforma, **quiero** `@org/ui-kit` compilado a `dist` y publicable, **para** que repos externos de miniapps lo instalen por versión.

## Aceptación
- `build` (tsc/tsup) emite `dist/` con JS + `.d.ts`; `main`/`types`/`exports` → `dist`; `files: ["dist"]`.
- peerDeps react/react-native se resuelven en el consumidor (no se empaquetan).
- `publishConfig` a GitHub Packages (como el contrato).
- El host sigue consumiéndolo (fuente en dev o publicado) — tsc + tests verdes.

## Notas
- Cuidar JSX/tipos RN en el build; no romper el consumo del host.
