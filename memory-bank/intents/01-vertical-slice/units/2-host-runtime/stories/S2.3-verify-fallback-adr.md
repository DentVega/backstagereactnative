# S2.3 — Verificación de manifest + fallback (con ADR)

**Como** host, **quiero** verificar la integridad del manifest y degradar con gracia, **para** no crashear ni montar código no confiable.

## Aceptación
- Verificación del manifest antes de montar (hash/firma — decidido por ADR).
- Fallback visible (sin crash) cuando: chunk inalcanzable, manifest inválido, o **skew** de singletons incompatible.
- Logs sin PII/secretos.
- ADR `adr-001-chunk-integrity.md` (context/decision/consequences).
- ≥1 test RNTL del fallback; unit tests de verify + detección de skew.

## Notas
- Host · sin nativos · núcleo de seguridad de federación.
