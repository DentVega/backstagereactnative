# S4.2 — GET /api/resolve (contrato con el host)

**Como** host, **quiero** un endpoint que resuelva id → {url, version, manifest}, **para** saber qué miniapp montar.

## Aceptación
- `GET /api/resolve?id=&host=` devuelve `ResolveResponse` válido (contrato Unit 1).
- Selección de versión (última publicada / rango compatible).
- Manifest validado antes de responder; error claro si no existe.
- Tests de API (resolve happy path + no encontrado + versión incompatible).

## Notas
- Web (Next.js) · consumido por Unit 2 · sin PII en logs.
