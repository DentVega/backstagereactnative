# S3.1 — Dominio: cuentas y movimientos (funciones puras)

**Como** miniapp, **quiero** un dominio testeable de cuentas/movimientos, **para** formatear saldos y movimientos sin acoplar a la UI.

## Aceptación
- Modelo `Account`, `Transaction` (value objects) framework-free.
- Funciones puras: formato de moneda, agrupación por fecha, cálculo de saldo.
- Unit tests (sin renderer) con casos borde (montos negativos, cero, distintos locales).

## Notas
- Remote (capa dominio) · sin nativos · TS strict, sin `any`.
