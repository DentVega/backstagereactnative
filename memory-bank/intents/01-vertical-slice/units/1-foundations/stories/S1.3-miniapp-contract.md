# S1.3 — packages/miniapp-contract

**Como** dev de miniapps y host, **quiero** un contrato tipado compartido, **para** que host y remotes acuerden manifest, entry, capabilities y shared list.

## Aceptación
- Tipos: `Manifest { id, version, entry, shared[], capabilities[], integrity? }`, `MiniappEntryProps { capabilities }`, `ResolveResponse { id, version, url, manifest }`.
- Importable desde host y miniapp (workspace).
- **Publicable como `@org/miniapp-contract`**: `name` con scope, `version` semver, `publishConfig` apuntando a npm privado / GitHub Packages, build de tipos (`dist/` + `.d.ts`). El repo de Backstage lo instalará por versión.
- `tsc` verde; sin dependencias de runtime pesadas.

## Notas
- Paquete publicado (único acoplamiento cross-repo) · sin nativos · todo cambio = bump de versión + changelog.
